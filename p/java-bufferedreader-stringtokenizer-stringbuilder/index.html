Coding Problem

자바 코딩문제 입력 기본기:
BufferedReader, StringTokenizer, StringBuilder

자바로 코딩 문제를 풀다 보면 거의 공식처럼 같이 나오는 조합이 있습니다. 입력은 빠르게 받고, 값을 나눠 읽고, 결과는 효율적으로 출력하기 위한 기본 조합입니다.

2026-03-11 Java / Input Basics BufferedReader, StringTokenizer, StringBuilder
BufferedReader StringTokenizer StringBuilder 왜 같이 쓰나 자주 쓰는 형태 핵심 정리

BufferedReader

  • 입력을 빠르게 받기 위한 클래스입니다.
  • `Scanner`보다 일반적으로 빠르기 때문에 코딩문제에서 자주 사용됩니다.
  • 한 줄 단위로 문자열을 읽으므로, 숫자로 쓰려면 직접 변환해야 합니다.
BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
int n = Integer.parseInt(br.readLine());

StringTokenizer

  • 공백이나 특정 구분자를 기준으로 문자열을 잘라 토큰으로 나눕니다.
  • `BufferedReader`로 읽은 한 줄을 숫자 여러 개로 분리할 때 많이 씁니다.
  • 예를 들어 `"10 20 30"`을 `10`, `20`, `30`으로 나눠서 꺼낼 수 있습니다.
StringTokenizer st = new StringTokenizer(br.readLine());
int a = Integer.parseInt(st.nextToken());
int b = Integer.parseInt(st.nextToken());

StringBuilder

  • 문자열을 효율적으로 이어붙이는 클래스입니다.
  • `String`에 `+`를 반복하면 객체가 계속 새로 만들어져 비효율적입니다.
  • 출력이 많을 때 결과를 `StringBuilder`에 모아서 마지막에 한 번 출력하면 성능이 좋습니다.
StringBuilder sb = new StringBuilder();
sb.append("Hello").append('\n');
sb.append(123);
System.out.print(sb);

코딩문제에서 함께 쓰는 이유

  • `BufferedReader`: 빠른 입력
  • `StringTokenizer`: 입력값 분리
  • `StringBuilder`: 빠른 출력 문자열 생성
흐름으로 보면 더 쉽습니다

한 줄을 빠르게 읽고, 필요한 값으로 나누고, 결과를 모아서 한 번에 출력하는 조합이라고 보면 됩니다.

자주 쓰는 형태

import java.io.*;
import java.util.*;

public class Main {
    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());
        StringBuilder sb = new StringBuilder();

        int a = Integer.parseInt(st.nextToken());
        int b = Integer.parseInt(st.nextToken());

        sb.append(a + b);
        System.out.print(sb);
    }
}

핵심 정리

  • 입력 속도는 `BufferedReader`
  • 입력 분해는 `StringTokenizer`
  • 출력 누적은 `StringBuilder`
