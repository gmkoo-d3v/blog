Java for-each 문:
언제 쓰고, 언제 쓰면 안 되는가

Java에서 배열이나 컬렉션을 순회할 때 `for-each` 문은 가장 읽기 쉬운 선택지입니다. 다만 인덱스가 필요하거나 순회 중 구조를 바꿔야 하는 상황에서는 오히려 맞지 않을 수 있습니다.

2026-03-11 Java / Language Basics Array, List, Set, Map
개요 기본 문법 일반 for문과 차이 배열에서의 동작 컬렉션에서의 동작 값 변경이 안 되는 이유 삭제할 때 주의할 점 언제 쓰고 언제 피해야 하나 예제 모음 정리

개요

Java의 `for-each` 문은 정확히는 `enhanced for statement`입니다. 핵심 목적은 "원소를 하나씩 꺼내서 읽는 코드"를 더 짧고 안전하게 만드는 것입니다.

일반 `for` 문은 강력하지만, 단순 순회에서도 직접 인덱스를 관리해야 합니다. 

for (int i = 0; i < list.size(); i++) {
    System.out.println(list.get(i));
}

같은 코드를 `for-each`로 쓰면 더 의도가 분명해집니다.

for (String item : list) {
    System.out.println(item);
}
핵심 기준

`for-each`는 "몇 번째 원소인가"보다 "현재 원소가 무엇인가"에 집중하는 문법입니다.

기본 문법

문법은 아래처럼 단순합니다.

for (타입 변수명 : 반복대상) {
    // 반복 실행 코드
}

예를 들어:

int[] numbers = {1, 2, 3};

for (int number : numbers) {
    System.out.println(number);
}

반복 대상에는 보통 아래가 들어갑니다.

  • 배열: `int[]`, `String[]`
  • `Iterable` 구현 컬렉션: `List`, `Set`, `Queue`

일반 for문과 차이

일반 `for` 문:

for (int i = 0; i < fruits.length; i++) {
    System.out.println(fruits[i]);
}

`for-each` 문:

for (String fruit : fruits) {
    System.out.println(fruit);
}

차이는 명확합니다.

  • 일반 `for`: 인덱스 접근 가능
  • `for-each`: 코드가 짧고 읽기 쉬움
  • 일반 `for`: 특정 위치 수정, 역순 순회, 앞뒤 비교에 유리
  • `for-each`: 단순 조회와 출력에 유리

배열에서의 동작

배열에서는 앞에서부터 끝까지 순서대로 원소를 꺼냅니다.

String[] fruits = {"apple", "banana", "grape"};

for (String fruit : fruits) {
    System.out.println(fruit);
}

실행 흐름은 다음과 같습니다.

  • 첫 번째 반복: `fruit = "apple"`
  • 두 번째 반복: `fruit = "banana"`
  • 세 번째 반복: `fruit = "grape"`

배열은 순서가 보장되므로 선언된 순서대로 출력됩니다.

컬렉션에서의 동작

`List`에서도 같은 방식으로 사용할 수 있습니다.

List<String> names = List.of("Kim", "Lee", "Park");

for (String name : names) {
    System.out.println(name);
}

`Set`에서도 사용할 수 있습니다.

Set<String> tags = Set.of("java", "spring", "jpa");

for (String tag : tags) {
    System.out.println(tag);
}

다만 `Set`은 구현체에 따라 순서가 일정하지 않을 수 있습니다. 그래서 `for-each`는 특히 인덱스가 필요 없는 컬렉션과 잘 맞습니다.

`Map`은 보통 `entrySet()`을 통해 순회합니다.

Map<String, Integer> scores = Map.of(
    "Kim", 90,
    "Lee", 80
);

for (Map.Entry<String, Integer> entry : scores.entrySet()) {
    System.out.println(entry.getKey() + " : " + entry.getValue());
}

값 변경이 안 되는 이유

아래 코드는 많은 초보자가 "배열 값이 바뀌겠지"라고 생각하지만 실제로는 바뀌지 않습니다. 

int[] numbers = {1, 2, 3};

for (int number : numbers) {
    number = number * 10;
}

이유는 `number`가 배열 원소 자체가 아니라 현재 원소 값을 복사한 지역 변수이기 때문입니다.

원본을 실제로 바꾸려면 인덱스 접근이 필요합니다.

for (int i = 0; i < numbers.length; i++) {
    numbers[i] = numbers[i] * 10;
}

참조형 객체는 왜 수정되는가

class User {
    String name;

    User(String name) {
        this.name = name;
    }
}

List<User> users = new ArrayList<>();
users.add(new User("A"));
users.add(new User("B"));

for (User user : users) {
    user.name = user.name + "_updated";
}

이 경우 객체 내부 상태는 바뀝니다. 참조 변수 자체를 다른 객체로 바꾸는 것은 원본 리스트에 반영되지 않지만, 참조가 가리키는 객체의 필드를 수정하면 실제 객체 상태는 변경됩니다.

삭제할 때 주의할 점

`for-each`를 돌면서 컬렉션을 직접 수정하면 `ConcurrentModificationException`이 발생할 수 있습니다. 

for (String name : names) {
    names.remove(name);
}

이유는 `for-each`가 내부적으로 `Iterator` 기반으로 동작하는데, 반복 도중 컬렉션 구조가 예상치 않게 바뀌면 반복 상태가 깨지기 때문입니다.

안전하게 삭제하려면 `Iterator`를 직접 사용하는 편이 낫습니다.

Iterator<String> iterator = names.iterator();

while (iterator.hasNext()) {
    String name = iterator.next();
    if (name.startsWith("K")) {
        iterator.remove();
    }
}

언제 쓰고 언제 피해야 하나

`for-each`를 쓰기 좋은 경우:

  • 전체 원소를 출력할 때
  • 합계나 통계를 계산할 때
  • 각 원소를 읽어 처리할 때
  • 인덱스가 전혀 필요 없을 때

일반 `for`가 더 좋은 경우:

  • 현재 몇 번째 원소인지 알아야 할 때
  • 특정 위치 값을 바꿔야 할 때
  • 앞 원소와 뒤 원소를 비교할 때
  • 역순 순회가 필요할 때
  • 반복 중 구조 변경을 해야 할 때
한 줄 기준

읽기 전용 순회면 `for-each`, 위치 제어가 필요하면 일반 `for`, 삭제까지 필요하면 `Iterator`를 우선 고려하면 됩니다.

예제 모음

배열 출력

int[] arr = {10, 20, 30};

for (int value : arr) {
    System.out.println(value);
}

List 문자열 순회

List<String> colors = List.of("red", "green", "blue");

for (String color : colors) {
    System.out.println(color.toUpperCase());
}

객체 리스트 순회

for (User user : users) {
    System.out.println(user.name);
}

Map 순회

for (Map.Entry<String, Integer> entry : scores.entrySet()) {
    System.out.println(entry.getKey() + " -> " + entry.getValue());
}

합계 계산

int sum = 0;

for (int value : numbers) {
    sum += value;
}

정리

`for-each` 문은 Java에서 가장 읽기 좋은 반복문 중 하나입니다. 배열과 컬렉션을 순회할 때 인덱스 실수를 줄여 주고, 코드 의도를 더 또렷하게 만듭니다.

  • `for-each` 변수 값을 바꿔도 원본 배열 원소는 바뀌지 않습니다.
  • 순회 중 컬렉션을 직접 삭제하면 예외가 날 수 있습니다.
  • 원소를 읽기만 하면 `for-each`, 위치 제어가 필요하면 일반 `for`가 더 적합합니다.
