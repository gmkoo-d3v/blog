보안 용어 한 번에 정리:
세션 하이재킹부터 XSS까지

보안 용어를 설명, 암기용 요약 표와 비교표까지 한 번에 정리.

2026-04-02 Security Basics 보안, 세션 하이재킹, XSS
개요 네트워크 공격 피싱·사회공학 웹·시스템 공격 시험 암기용 요약 표 표 형태 비교 정리 정리

개요

보안 용어는 이름만 보면 비슷해 보여도, 실제로는 무엇을 노리는가에 따라 구분하면 훨씬 쉽게 외울 수 있다.

이번 글은 네트워크 공격, 사람을 속이는 공격, 웹 공격, 데이터 관리 개념으로 나눠서 정리했다.

빠른 암기 기준

네트워크를 노리면 세션 하이재킹, ARP 스푸핑, 스니핑 쪽이고, 사람을 속이면 스미싱, 스피어 피싱, 사회공학 쪽으로 먼저 묶어두면 덜 헷갈린다.

네트워크 공격

세션 하이재킹(Session Hijacking)

사용자가 정상적으로 로그인한 뒤 만들어진 세션을 공격자가 가로채, 원래 사용자처럼 서버 자원에 접근하는 공격이다.

즉, 비밀번호를 직접 몰라도 인증이 끝난 연결 상태를 뺏으면 공격이 가능하다는 점이 핵심이다.

TCP 세션 하이재킹

TCP 통신 과정에 끼어들어 클라이언트와 서버 사이의 시퀀스 번호를 가로채는 방식이다. 보통 3-way handshake와 동기화된 시퀀스 번호가 시험 포인트로 함께 나온다.

ARP 스푸핑(Address Resolution Protocol spoofing)

ARP의 약점을 이용해 공격자가 자신의 MAC 주소를 마치 다른 장비의 주소인 것처럼 속이는 공격이다. 이렇게 되면 원래 피해자에게 가야 할 패킷을 가로채거나 통신을 방해할 수 있다.

용어 의미
MAC 네트워크 장비의 물리적 주소
ARP IP 주소를 MAC 주소로 변환
RARP MAC 주소를 IP 주소로 변환

스니핑(Sniffing)

네트워크 중간에서 오가는 패킷 정보를 몰래 엿듣는 공격이다. 직접 변조하지 않고 감청하는 방식이라 수동적 공격으로 분류된다.

피싱·사회공학

스미싱(Smishing)

문자메시지(SMS)로 링크 클릭이나 악성 앱 설치를 유도해 개인정보를 탈취하는 공격이다. 최근에는 택배, 청첩장, 경품 안내 같은 자연스러운 문구로 접근하는 경우가 많다.

사회공학(Social Engineering)

시스템의 기술적 취약점을 직접 공격하기보다, 사람의 심리와 신뢰를 이용해 정상 보안 절차를 무너뜨리는 비기술적 공격 방식이다.

스피어 피싱(Spear Phishing)

특정 개인이나 조직을 목표로 하는 표적형 피싱이다. 실제 업무 메일처럼 위장한 이메일을 보내 링크 클릭이나 첨부파일 실행을 유도한다.

큐싱(Qshing)

QR코드를 이용해 악성 앱 설치나 피싱 사이트 접속을 유도하는 금융사기 기법이다. 문자 기반이면 스미싱, QR 기반이면 큐싱으로 구분해 외우면 편하다.

타이포스쿼팅(Typosquatting)

유명 사이트와 비슷한 오타 도메인을 미리 등록해 사용자의 오입력을 노리는 방식이다. URL 하이재킹이라는 표현으로도 자주 나온다.

웹·시스템 공격

APT(Advanced Persistent Threat)

특정 기업이나 조직에 장기간 잠복하면서 지속적으로 공격하는 지능형 위협이다. 침투 후 바로 끝나는 것이 아니라, 내부 거점을 마련해 정보를 수집하고 외부로 유출하는 흐름이 특징이다.

무작위 대입 공격(Brute Force Attack)

가능한 값을 전부 시도해 비밀번호나 암호키를 찾아내는 공격이다. 단순하지만 비밀번호가 약할수록 매우 치명적이다.

SQL 인젝션(SQL Injection)

입력값 검증 취약점을 이용해 SQL문을 주입하고 데이터베이스를 조회, 수정, 삭제하는 공격이다. 웹 입력창을 통해 DB를 직접 노린다고 이해하면 쉽다.

XSS(Cross Site Scripting)

웹페이지에 악성 스크립트를 삽입해 사용자의 브라우저에서 실행되도록 만드는 공격이다. 서버 자체보다 사용자 브라우저가 공격 실행 지점이라는 점이 핵심이다.

다크 데이터(Dark Data)

특정 목적으로 수집했지만 이후 활용되지 않고 저장만 되어 있는 데이터다. 공격 기법은 아니지만, 관리되지 않은 데이터가 늘어날수록 비용과 보안 위험도 함께 커진다.

데이터 관련 개념 다크 데이터 공격 기법이 아니라 관리 대상 개념

정리

  • 세션 하이재킹은 인증 후 세션을 빼앗는 공격이다.
  • ARP 스푸핑은 MAC 주소를 속여 패킷 흐름을 가로챈다.
  • 스미싱과 큐싱은 각각 문자와 QR코드를 이용한 피싱이다.
  • 스피어 피싱은 특정 대상을 정한 표적형 이메일 공격이다.
  • SQL 인젝션은 DB를, XSS는 브라우저를 직접 노린다.
  • 다크 데이터는 공격이 아니라 활용되지 않은 저장 데이터 개념이다.
