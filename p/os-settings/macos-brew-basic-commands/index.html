OS 설정

맥 앱 관리가 낯설다면, Brew 기본 명령어부터 익히기

Windows와 다른 macOS 앱 관리 방식에 익숙해지고 싶을 때, Homebrew에서 가장 먼저 손에 익히면 좋은 기본 명령어를 한 번에 정리했습니다.

2026-03-12 macOS / Homebrew 입문용 커맨드 정리
개요 Brew가 뭐냐면 설치 제거 업데이트 검색과 목록 확인 정리와 점검 Brewfile로 한 번에 설치 마무리

개요

Windows에 익숙한 사람이라면 프로그램을 설치할 때 보통 웹사이트에 들어가 설치 파일을 내려받고, 실행해서 직접 설치하는 흐름이 자연스럽습니다. 반면 macOS에서는 이런 방식과 함께 Homebrew 같은 패키지 관리 도구를 사용하는 문화가 꽤 익숙하게 자리 잡아 있습니다.

처음 맥을 쓰기 시작하면 터미널로 앱을 설치하고 관리한다는 개념이 조금 낯설 수 있습니다. 하지만 brew를 익혀두면 앱 설치, 삭제, 업데이트, 정리를 훨씬 더 단순하고 일관되게 처리할 수 있습니다.

이 글의 목표

복잡한 옵션을 한꺼번에 다루기보다, 맥에서 앱 관리 흐름에 익숙해지기 위해 가장 자주 쓰는 기본 커맨드만 먼저 손에 익히는 데 초점을 맞췄습니다.

Brew가 뭐냐면

Homebrew는 macOS에서 패키지와 앱을 설치하고 관리할 수 있게 해주는 도구입니다. 터미널 명령어만으로 원하는 항목을 설치하거나, 필요 없는 앱을 지우고, 전체 목록을 확인하고, 업데이트까지 진행할 수 있습니다.

쉽게 말해 맥에서 앱 관리 흐름을 조금 더 체계적으로 바꿔주는 도구라고 보면 됩니다.

설치

가장 먼저 익히게 되는 명령어는 install입니다. 터미널에서 사용하는 패키지는 일반 설치 방식으로, macOS 앱은 --cask 옵션을 붙여 설치합니다. 

brew install wget
brew install --cask google-chrome
  • brew install [패키지]
  • brew install --cask [앱]

예를 들어 wget처럼 명령줄에서 쓰는 도구는 install로 설치하고, 크롬처럼 일반 앱은 --cask를 붙여 설치하면 됩니다.

제거

설치한 항목이 더 이상 필요 없으면 uninstall로 삭제할 수 있습니다. 

brew uninstall wget
brew uninstall --cask google-chrome --zap
  • brew uninstall [패키지]
  • brew uninstall --cask [앱] --zap

특히 앱을 지울 때 붙는 --zap 옵션은 꽤 유용합니다. 앱 자체만 삭제하는 것이 아니라 관련 설정 파일까지 함께 정리할 때 도움이 됩니다.

업데이트

brew는 한 번 설치하고 끝나는 도구가 아니라 계속 관리하면서 쓰게 됩니다. 그래서 업데이트 관련 명령어도 자주 사용하게 됩니다. 

brew update
brew upgrade
brew upgrade node
  • brew update
  • brew upgrade
  • brew upgrade [패키지]

brew update는 Homebrew 자체의 정보를 최신 상태로 갱신하고, brew upgrade는 설치된 패키지나 앱을 최신 버전으로 올리는 역할을 합니다.

검색과 목록 확인

무언가를 설치하고 싶을 때 정확한 이름이 기억나지 않거나, 지금 내 맥에 무엇이 설치되어 있는지 확인하고 싶을 때 아래 명령어를 사용하면 됩니다. 

brew search chrome
brew info node
brew list
brew list --cask
  • brew search [키워드]
  • brew info [패키지]
  • brew list
  • brew list --cask

brew search는 설치 가능한 항목을 찾을 때 쓰고, brew info는 버전, 설치 정보, 의존성 같은 세부 내용을 보여줍니다. brew list와 brew list --cask는 현재 설치된 패키지와 앱을 각각 확인할 때 유용합니다.

정리와 점검

오래 사용하다 보면 캐시나 이전 버전이 쌓일 수 있고, 가끔은 환경 상태를 점검하고 싶을 때도 있습니다. 그럴 때는 아래 명령어를 같이 알아두면 좋습니다. 

brew cleanup
brew doctor
brew deps python
  • brew cleanup
  • brew doctor
  • brew deps [패키지]

brew cleanup은 오래된 캐시나 불필요한 파일을 정리할 때 사용하고, brew doctor는 현재 Homebrew 환경에 문제가 없는지 점검할 때 유용합니다. brew deps는 특정 패키지가 어떤 의존성을 가지는지 확인할 때 사용할 수 있습니다.

Brewfile로 한 번에 설치

앱을 하나씩 설치하는 방식에 익숙해진 뒤에는 Brewfile로 여러 앱을 한 번에 설치하는 흐름도 같이 알아두면 편합니다. 새 맥을 세팅하거나 자주 쓰는 앱 조합을 다시 맞출 때 특히 유용합니다.

예를 들어 웹브라우저와 필수 유틸 정도를 한 번에 설치하고 싶다면 아래처럼 정리할 수 있습니다. 

cask "google-chrome"
cask "firefox"
cask "naver-whale"
cask "brave-browser"
cask "karabiner-elements"

이 내용을 Brewfile이라는 파일에 저장한 뒤 아래처럼 실행하면 됩니다. 

brew bundle --file=./Brewfile

즉, brew install --cask를 여러 번 반복하는 대신 자주 쓰는 앱 목록을 파일로 관리해두고 한 번에 설치하는 방식입니다. 맥을 새로 세팅할 때도 훨씬 편하고, 어떤 앱을 기본으로 쓰는지 스스로 정리하기도 좋습니다.

마무리

맥은 Windows와 앱 관리 방식이 조금 다르게 느껴질 수 있습니다. 그래서 처음에는 불편하게 느껴질 수 있지만, brew에 익숙해지면 설치, 삭제, 업데이트, 정리까지 훨씬 더 단순하게 관리할 수 있습니다.

처음부터 모든 명령어를 외울 필요는 없습니다. 아래 명령어부터 자연스럽게 익히는 것을 목표로 해도 충분합니다. 

brew install
brew uninstall
brew update
brew upgrade
brew search
brew list
brew bundle --file=./Brewfile

맥이 아직 낯설다면, 복잡하게 생각하지 말고 필요한 앱 하나를 직접 brew로 설치해보는 것부터 시작해보면 됩니다.
