실리콘 맥(도커)에서 하는방법 Oracle 11g 실습하는 방법

실리콘 맥에서 Oracle 11g 실습 환경을 만들 때는 직접 설치보다 Docker가 훨씬 덜 피곤하다. 이 글은 Oracle XE 11g 실행부터 sqlplus 접속, SCOTT 생성, 샘플 테이블 입력, 추가계정에 복제까지 한 페이지에서 끝나게 정리했다.

2026-04-18 Apple Silicon · Docker Oracle XE 11g
개요 준비물 기존 컨테이너 정리 이미지 다운로드 컨테이너 실행 로그 확인 sqlplus 접속 SCOTT 생성 SCOTT 테이블 생성 추가계정 생성 추가계정 복제 최종 확인 DBeaver 연결 마무리

개요

실리콘 맥에서 Oracle 11g를 바로 설치하려고 하면 버전 호환과 아키텍처 문제로 생각보다 시간을 많이 쓰게 된다. 특히 Apple Silicon 환경은 예전 x86 기준 설치 흐름과 잘 맞지 않는 경우가 많아서, 실습 목적이라면 Docker로 먼저 띄우는 편이 훨씬 단순하다.

이번 글의 목표는 "실습 환경을 한 번에 재현하는 것"이다. 그래서 컨테이너만 띄우고 끝내지 않고, 바로 SCOTT 계정을 만들고 대표 샘플 테이블인 EMP, DEPT, BONUS, SALGRADE까지 넣은 뒤, 같은 구조를 추가계정 계정으로 복제하는 흐름까지 이어서 간다.

핵심 포인트

실리콘 맥에서는 --platform linux/amd64 옵션이 사실상 핵심이다. 이 옵션이 빠지면 Oracle 11g 이미지 실행 단계에서부터 꼬일 수 있다.

그리고 이 글에서는 예시 비밀번호를 모두 password로 통일했다. 실습이 끝나면 본인 환경에 맞는 비밀번호로 바꿔 쓰면 된다.

먼저 준비할 것

시작하기 전에 Docker Desktop이 설치되어 있어야 한다. 아직 설치하지 않았다면 Docker Desktop부터 먼저 올려두고, 터미널에서 Docker가 정상 동작하는지만 한 번 확인하면 된다. 

docker --version

버전 정보가 잘 나오면 준비는 끝이다. 이제 Oracle 11g 실습용 컨테이너를 띄우면 된다.

이번 글에서 맞출 최종 구조

SCOTT은 교재용, 추가 계정는 부트캠프 실습용으로 두고, 두 계정 모두 같은 샘플 데이터를 보게 만드는 구성을 기준으로 설명한다.

1. 기존 Oracle 컨테이너가 있다면 정리하기

예전에 테스트하다가 같은 이름의 컨테이너를 만들어둔 경우가 있다. 이런 경우에는 이름 충돌 때문에 실행이 깔끔하지 않을 수 있어서 먼저 정리하고 가는 편이 좋다. 

docker rm -f oracle-xe 2>/dev/null

이미 컨테이너가 없으면 그냥 조용히 넘어가고, 있으면 강제로 삭제한다. 실습 환경을 처음부터 다시 맞출 때 자주 쓰는 한 줄이다.

2. Oracle XE 11g 이미지 내려받기

다음으로 실습에 사용할 이미지를 받아온다. 여기서는 gvenzl/oracle-xe:11-slim 태그를 기준으로 진행한다. 

docker pull gvenzl/oracle-xe:11-slim

처음 받는 경우에는 이미지 다운로드에 조금 시간이 걸릴 수 있다. 한 번 받아두면 다음부터는 훨씬 빠르게 재사용할 수 있다.

3. 실리콘 맥에서 Oracle 11g 컨테이너 실행하기

이제 실제로 컨테이너를 띄운다. 실리콘 맥에서는 아래 명령처럼 amd64 플랫폼을 명시하는 부분을 꼭 넣어야 한다. 

docker run -d \
  --platform linux/amd64 \
  --name oracle-xe \
  -p 1521:1521 \
  -e ORACLE_PASSWORD=password \
  gvenzl/oracle-xe:11-slim

옵션을 짧게 보면 아래 정도만 기억하면 된다.

  • --platform linux/amd64: Apple Silicon에서 x86 이미지 실행
  • --name oracle-xe: 컨테이너 이름 지정
  • -p 1521:1521: Oracle 기본 포트 연결
  • -e ORACLE_PASSWORD=password: system 계정 비밀번호 지정
포트가 이미 사용 중이라면

로컬에서 이미 1521 포트를 쓰는 Oracle이나 다른 DB가 떠 있다면 충돌이 날 수 있다. 그럴 때는 기존 프로세스를 내리거나, 실습용으로 호스트 포트를 다른 값으로 바꿔서 실행하면 된다.

4. DB가 다 올라왔는지 로그로 확인하기

컨테이너가 실행됐다고 해서 바로 접속 가능한 것은 아니다. Oracle은 기동에 시간이 조금 걸리기 때문에 로그를 보면서 준비가 끝나는 시점을 확인하는 편이 안전하다. 

docker logs -f oracle-xe

로그가 계속 흘러가다가 데이터베이스 준비가 끝나는 흐름이 보이면 그때 접속하면 된다. 처음 실행에서는 몇 분 정도 걸릴 수 있으니 조급하게 다시 띄우기보다 잠깐 기다리는 편이 낫다.

5. sqlplus로 접속하기

컨테이너가 다 올라왔으면 이제 내부로 들어가서 sqlplus에 접속한다. 여기서부터는 Oracle 안에서 계정과 테이블을 세팅하면 된다. 

docker exec -it oracle-xe bash
sqlplus system/password

정상적으로 접속되면 Oracle 11g 실습용 서버가 잘 올라온 것이다. 이제 바로 SCOTT 계정부터 만들어보면 된다.

6. SCOTT 계정 만들기

교재 예제에서 가장 많이 쓰는 계정이 SCOTT이다. 먼저 system 계정으로 접속한 상태에서 아래 명령을 실행한다. 

CREATE USER scott IDENTIFIED BY tiger;
GRANT CONNECT, RESOURCE TO scott;
ALTER USER scott ACCOUNT UNLOCK;
GRANT UNLIMITED TABLESPACE TO scott;

여기까지 하면 SCOTT 계정 자체는 만들어진다. 이제 실제 실습용 테이블과 데이터를 넣어주면 된다.

7. SCOTT 테이블과 샘플 데이터 넣기

아래 SQL은 대표적인 Oracle 실습용 샘플 구조다. DEPT를 먼저 만들고, 그다음 EMP, BONUS, SALGRADE를 순서대로 넣으면 된다.

CREATE TABLE dept (
  deptno NUMBER(2) PRIMARY KEY,
  dname VARCHAR2(14),
  loc VARCHAR2(13)
);

  sal NUMBER(7,2),

순서가 중요한 이유

EMP는 DEPT를 참조하므로, 부서 테이블을 먼저 만들고 데이터를 넣은 뒤 사원 테이블을 생성하는 편이 가장 깔끔하다.

8. 추가 계정 만들기

이제 실습용 계정을 하나 더 만든다. 실습 편의를 위해 비밀번호는 password로 맞춘다. 

CONNECT system/password;

CREATE USER 추가 계정 IDENTIFIED BY "password";
GRANT CONNECT, RESOURCE TO 추가 계정;
ALTER USER 추가 계정 ACCOUNT UNLOCK;
GRANT UNLIMITED TABLESPACE TO 추가 계정;

9. SCOTT 데이터를 추가아이디로 복제하기

여기까지 왔으면 이제 핵심 단계다. 교재는 SCOTT 기준으로 풀고, 부트캠프 실습은 추가계정 기준으로 풀 수 있게 데이터를 그대로 복제한다. 

CONNECT 추가계정/"password";

CREATE TABLE dept AS SELECT * FROM scott.dept;
CREATE TABLE emp AS SELECT * FROM scott.emp;
CREATE TABLE bonus AS SELECT * FROM scott.bonus;
CREATE TABLE salgrade AS SELECT * FROM scott.salgrade;

ALTER TABLE dept ADD PRIMARY KEY (deptno);
ALTER TABLE emp ADD PRIMARY KEY (empno);
ALTER TABLE emp ADD FOREIGN KEY (deptno) REFERENCES dept(deptno);
하나만 기억할 점

CREATE TABLE ... AS SELECT ...는 데이터를 빠르게 복사할 때는 좋지만, 모든 제약조건을 자동으로 복사하지는 않는다. 그래서 실습용으로 구조를 최대한 맞추려면 뒤에 PK/FK를 다시 추가해주는 편이 좋다.

10. 최종 확인

마지막으로 SCOTT과 추가계정의 데이터 건수가 같은지만 보면 실습 환경이 거의 맞춰졌다고 보면 된다. 

CONNECT scott/tiger;
SELECT COUNT(*) FROM emp;

CONNECT 추가계정/"password";
SELECT COUNT(*) FROM emp;

정상이라면 둘 다 14가 나온다. 이 숫자가 같으면 최소한 EMP 데이터 복제는 제대로 된 것이다.

11. DBeaver 연결 정보

터미널에서 확인이 끝났다면 이제 GUI 툴로 붙여서 쓰면 훨씬 편하다. DBeaver에서는 아래처럼 연결하면 된다.

복제된 계정

마무리

실리콘 맥에서 Oracle 11g 실습 환경을 만들 때는 직접 설치에 매달리기보다 Docker로 먼저 띄우는 쪽이 훨씬 현실적이다. 특히 --platform linux/amd64만 정확히 챙기면 설치 흐름 자체는 생각보다 단순하다.

순서를 한 줄로 줄이면 이렇다. Docker로 Oracle XE 11g 실행, sqlplus 접속, SCOTT 생성, 샘플 테이블 입력, 추가계정 생성, 그리고 데이터 복제. 이렇게 맞춰두면 교재용 계정과 실습용 계정을 깔끔하게 분리하면서도 같은 데이터를 기준으로 연습할 수 있다.
