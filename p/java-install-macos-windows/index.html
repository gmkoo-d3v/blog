Java 설치 가이드: macOS(Homebrew)와 Windows 환경 변수 설정

macOS와 Windows에서 Java를 설치하고, 버전 확인과 `JAVA_HOME` 설정까지 바로 이어서 할 수 있게 정리했습니다.

2026-03-10 macOS / Windows Java 17, 21 예시
개요 macOS 설치 macOS zsh 설정 macOS bash 설정 Windows 설치 검증

개요

Java를 설치할 때 중요한 것은 “설치 자체”보다 “버전을 명확하게 관리할 수 있느냐”입니다. 프로젝트마다 Java 17, 21처럼 요구 버전이 달라질 수 있기 때문에, 설치 직후에 `JAVA_HOME`과 `Path` 기준을 같이 잡아 두는 편이 안전합니다.

정리 기준

macOS는 Homebrew와 shell 설정으로, Windows는 설치 후 시스템 환경 변수 등록으로 정리했습니다.

macOS에서 설치하기

첨부된 기존 글의 흐름은 유지하되, 예전 `adoptopenjdk` 중심 설명은 걷어내고 Homebrew `openjdk` 공식 formula 기준으로 정리했습니다.

1. Homebrew 준비

Homebrew가 이미 있더라도 먼저 업데이트를 한 번 실행합니다.

brew update

2. 설치 가능한 OpenJDK 확인

brew search openjdk

보통은 LTS 버전 위주로 `openjdk@17`, `openjdk@21` 중 하나 또는 둘 다 설치해 두는 편이 실무에서 무난합니다.

3. 원하는 버전 설치

brew install openjdk@17
brew install openjdk@21

4. macOS가 JDK를 인식할 수 있게 링크 생성

sudo ln -sfn "$(brew --prefix openjdk@17)/libexec/openjdk.jdk" \
  /Library/Java/JavaVirtualMachines/openjdk-17.jdk

sudo ln -sfn "$(brew --prefix openjdk@21)/libexec/openjdk.jdk" \
  /Library/Java/JavaVirtualMachines/openjdk-21.jdk

이 작업을 해두면 `/usr/libexec/java_home -V`로 설치된 버전을 명확하게 조회할 수 있습니다.

5. 설치된 버전 확인

/usr/libexec/java_home -V
java -version

macOS: zsh 사용자는 ~/.zshrc

zsh 기준으로는 아래처럼 버전별 `JAVA_HOME`을 먼저 만들어 두고, 마지막에 실제 사용할 `JAVA_HOME`을 선택하면 됩니다.

TextEdit로 열려면 아래처럼 실행합니다.

touch ~/.zshrc
open -e ~/.zshrc
# ~/.zshrc
export JAVA_17_HOME=$(/usr/libexec/java_home -v 17)
export JAVA_21_HOME=$(/usr/libexec/java_home -v 21)

# 기본 버전 선택
export JAVA_HOME="$JAVA_21_HOME"
export PATH="$JAVA_HOME/bin:$PATH"

변경 사항 반영:

source ~/.zshrc
java -version

macOS: bash 사용자는 ~/.bashrc

bash도 구조는 동일합니다. 요구사항에 맞게 `~/.bashrc` 기준으로 정리하면 됩니다.

TextEdit로 열려면 아래처럼 실행합니다.

touch ~/.bashrc
open -e ~/.bashrc
# ~/.bashrc
export JAVA_17_HOME=$(/usr/libexec/java_home -v 17)
export JAVA_21_HOME=$(/usr/libexec/java_home -v 21)

# 기본 버전 선택
export JAVA_HOME="$JAVA_17_HOME"
export PATH="$JAVA_HOME/bin:$PATH"

변경 사항 반영:

source ~/.bashrc
java -version

버전 전환 요령

예를 들어 Java 17을 쓰다가 21로 바꾸고 싶으면 `JAVA_HOME`이 가리키는 대상만 바꾸면 됩니다. 

export JAVA_HOME="$JAVA_21_HOME"
export PATH="$JAVA_HOME/bin:$PATH"

Windows에서 설치하기

Windows는 Homebrew 대신 JDK 설치 프로그램을 사용하고, 설치 후 시스템 환경 변수에 `JAVA_HOME`과 `Path`를 등록하는 흐름으로 가는 것이 가장 단순합니다.

1. JDK 설치

Temurin, Oracle JDK 같은 배포판 중 하나를 선택해 Java 17 또는 21을 설치합니다. 설치 경로 예시는 아래처럼 잡히는 경우가 많습니다. 

C:\Program Files\Java\jdk-21

2. 시스템 환경 변수 창 열기

가장 쉬운 방법은 시작 메뉴에서 검색하는 것입니다.

시작 메뉴 → "환경 변수 편집" 검색 → "시스템 환경 변수 편집" 실행

또는 실행 창으로 바로 들어갈 수도 있습니다.

Win + R
sysdm.cpl

시스템 속성 창이 열리면 고급 탭으로 이동한 뒤 환경 변수(N)... 버튼을 누릅니다.

3. 시스템 환경 변수 등록

여러 버전을 관리할 예정이면 버전별 변수와 기본 `JAVA_HOME`을 같이 두는 편이 나중에 바꾸기 쉽습니다. 

JAVA_17_HOME = C:\Program Files\Java\jdk-17
JAVA_21_HOME = C:\Program Files\Java\jdk-21
JAVA_HOME    = %JAVA_21_HOME%

4. Path에 Java 실행 파일 추가

%JAVA_HOME%\bin

이미 이전 Java 경로가 `Path`에 직접 들어가 있다면 제거하고, `%JAVA_HOME%\bin` 하나만 남겨 두는 편이 충돌을 줄입니다.

5. 새 터미널에서 확인

echo %JAVA_HOME%
where java
java -version
javac -version

6. 버전 바꾸기

기본 버전을 바꾸고 싶으면 `JAVA_HOME`만 다른 버전 변수로 바꾸면 됩니다. 

JAVA_HOME = %JAVA_17_HOME%

검증 체크리스트

  • `java -version`이 기대한 버전을 출력하는지 확인한다.
  • `javac -version`도 같은 계열 버전을 출력하는지 확인한다.
  • macOS에서는 `/usr/libexec/java_home -V`로 설치 목록을 확인한다.
  • Windows에서는 `where java`가 `JAVA_HOME` 기준 경로를 가리키는지 본다.
핵심 포인트

Java 설치는 금방 끝나지만, 실제 문제는 대부분 `JAVA_HOME`과 `Path`가 꼬일 때 발생합니다. 설치 직후 바로 검증 명령을 실행해 두면 이후 셋업 비용이 크게 줄어듭니다.
