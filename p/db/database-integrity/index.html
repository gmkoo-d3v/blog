데이터베이스 무결성 정리:
개체·도메인·참조 무결성 한 번에 암기하기

반복해서 보기 좋은 형태로, 데이터베이스 무결성의 개념부터 핵심 3대 무결성, 전체 종류, 예시, 초압축 암기표까지 한 흐름으로 정리했다.

2026-04-02 DB / Integrity 데이터베이스, 무결성
개요 무결성 제약조건 핵심 3대 무결성 무결성 종류 총정리 비교 정리 예시 암기 표 정리

개요

데이터베이스에서 무결성(Integrity) 이란 저장된 데이터가 현실 세계의 실제 값과 일치하도록 정확성과 일관성을 유지하는 성질을 말한다.

쉽게 말하면 데이터가 틀리지 않고, 서로 모순되지 않으며, 정해진 규칙에 맞게 저장되어야 한다는 뜻이다.

시험 포인트

무결성 문제는 보통 기본키, 외래키, 속성값 범위를 묻는 형태로 나온다. 그래서 개체 무결성, 도메인 무결성, 참조 무결성 세 가지를 먼저 정확히 구분하는 것이 중요하다.

무결성 제약조건이란?

무결성 제약조건은 데이터베이스에 잘못된 데이터가 저장되는 것을 막기 위한 규칙이다. 이를 통해 데이터의 정확성, 일관성, 신뢰성을 유지할 수 있다.

구분 의미
무결성 데이터가 올바른 상태를 유지하는 성질
무결성 제약조건 그 올바른 상태를 지키기 위해 DB에 설정하는 규칙

무결성이 필요한 이유

이유 설명
정확성 보장 잘못된 데이터 입력을 막아 결과의 신뢰도를 높인다.
일관성 유지 데이터끼리 서로 충돌하거나 모순되는 상황을 방지한다.
업무 규칙 반영 현실 세계의 조건을 데이터베이스 구조에 그대로 적용한다.
오류 예방 조회, 수정, 삭제 과정에서 발생할 수 있는 논리 오류를 줄인다.

핵심 3대 무결성

시험에서는 보통 개체 무결성, 도메인 무결성, 참조 무결성을 가장 중요하게 다룬다.

종류 핵심 내용 암기 포인트
개체 무결성 기본키는 NULL 불가, 중복 불가 기본키 제약
도메인 무결성 속성값은 정해진 범위와 형식 안에서만 허용 값의 범위 제약
참조 무결성 외래키는 NULL 이거나 참조 기본키 값과 일치 외래키 제약

핵심 문장으로 바로 외우기

  • 개체 무결성: 기본키는 비어 있으면 안 되고, 중복되면 안 된다.
  • 도메인 무결성: 속성값은 정해진 값의 범위를 벗어나면 안 된다.
  • 참조 무결성: 외래키는 존재하지 않는 기본키 값을 참조하면 안 된다.

무결성 종류 총정리

종류 의미 핵심 키워드 예시
개체 무결성
Entity Integrity		 기본키를 구성하는 속성은 NULL 이거나 중복될 수 없다. 기본키, 유일성 학생 테이블의 학번
도메인 무결성
Domain Integrity		 속성값은 미리 정의된 범위, 형식, 자료형 안에 있어야 한다. 값의 범위, 형식 제한 학년은 1~4만 허용
참조 무결성
Referential Integrity		 외래키 값은 NULL 이거나 참조 테이블의 기본키와 같아야 한다. 외래키, 참조 관계 수강 테이블의 학번은 학생 테이블의 학번 참조
사용자 정의 무결성
User-Defined Integrity		 사용자가 정한 업무 규칙을 만족해야 한다. 업무 규칙 최대 신청 학점 21학점 이하
NULL 무결성 특정 속성은 NULL 값을 가질 수 없다. 필수 입력 이름, 학번은 반드시 입력
고유 무결성
Unique Integrity		 특정 속성값은 각 튜플마다 서로 달라야 한다. 중복 금지 이메일 중복 불가
키 무결성
Key Integrity		 하나의 릴레이션에는 적어도 하나의 키가 존재해야 한다. 식별 가능성 각 행을 구별할 키 필요
관계 무결성
Relationship Integrity		 릴레이션 간 관계가 논리적으로 타당하게 유지되어야 한다. 관계의 적절성 부모-자식 릴레이션 관계 유지

헷갈리는 개념 비교 정리

무결성 무엇을 검사하는가? 한 줄 암기
개체 무결성 기본키 자체가 올바른가 기본키는 비어 있거나 중복되면 안 됨
도메인 무결성 속성값의 범위와 형식이 올바른가 정해진 값만 입력 가능
참조 무결성 테이블 간 연결 관계가 올바른가 외래키는 실제 존재하는 값만 참조
빠른 구분법

기본키가 보이면 개체 무결성, 입력값 범위가 보이면 도메인 무결성, 외래키와 참조가 보이면 참조 무결성으로 먼저 연결해서 생각하면 문제 풀이가 쉬워진다.

예시로 정리

학생 테이블과 수강 테이블이 있다고 가정하면, 세 가지 핵심 무결성을 한 번에 이해할 수 있다.

학생 테이블

학번 이름 학년

2024002 이영희 2

수강 테이블

학번 과목명
2024001 데이터베이스

상황 해당 무결성
학생 테이블의 학번은 중복되거나 NULL일 수 없다. 개체 무결성
학생 테이블의 학년은 1, 2, 3, 4만 가능하다. 도메인 무결성
수강 테이블의 학번은 학생 테이블에 존재하는 값이어야 한다. 참조 무결성

암기용 초압축 표

종류 암기 문장
개체 무결성 기본키는 NULL 불가, 중복 불가
도메인 무결성 속성값은 정해진 범위 내 값만 허용
참조 무결성 외래키는 NULL 또는 참조 기본키 값
사용자 정의 무결성 업무 규칙은 사용자가 직접 정함
NULL 무결성 특정 속성은 NULL 금지
고유 무결성 특정 속성값 중복 금지
키 무결성 릴레이션에는 최소 1개 이상의 키 필요
관계 무결성 릴레이션 간 관계의 타당성 유지

정리

무결성은 데이터베이스의 데이터가 정확하고 일관되게 유지되도록 하는 성질이며, 이를 보장하기 위한 규칙이 무결성 제약조건이다.

개체 무결성, 도메인 무결성, 참조 무결성 세 가지를 먼저 확실하게 암기하고, 나머지 무결성은 보조 개념으로 연결해 두면 정리가 훨씬 쉬워진다.
